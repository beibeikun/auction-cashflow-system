# auction-app 新手快速使用说明

这是一份给第一次使用者看的短说明。只要按顺序做，就可以完成一场拍卖的基本录入、结算和导出。

## 1. 先记住 5 件事

1. 价格按千单位填：`8` 代表 `8,000`。
2. 流拍时买家号牌填 `-1`。
3. 正常成交用 `收` 和 `付` 确认。
4. 流拍或顶价退回用 `退` 确认。
5. 每天结束要备份 `data/store.json` 和 `data/sessions/`。

## 2. 打开系统

主机电脑先启动：

```bash
cd auction-app
npm start
```

本机打开：

```text
http://localhost:4173
```

其他电脑打开终端显示的局域网地址，例如：

```text
http://192.168.1.10:4173
```

## 3. 开始新场次

进入 `场次设置`：

1. 在 `拍卖场次管理` 新建空白场次，或切换到已有场次。
2. 填拍卖会名称。
3. 检查开拍时间。
4. 检查默认出货佣金和买货佣金。
5. 如需修改传票抬头，在 `公司资料` 填税号、邮编、地址和 Logo。
6. 点击 `保存设置` 或 `保存公司资料`。

如果要清空上一场成交和客户：

```text
场次设置 -> 清空本场拍卖与客户数据
```

这个按钮会清空当前场次的成交和本场客户登记，不会删除共享永久客户簿和全场次共享代码表。

## 4. 登记客户

进入 `客户资料`。

如果客户已经在 `永久客户簿列表` 中：

1. 可以先用 `搜索客户簿` 查找实际名称、电话、地址或古物商证编号。
2. 点击该客户左侧的 `引用`。
3. 上方 `本场客户登记` 会自动带入客户资料。
4. 填本场的买货号牌、出货号牌和佣金后，点击 `新增登记`。

如果是新客户，在上方 `本场客户登记` 直接填写。最少要填：

| 字段 | 例子 |
|---|---|
| 买货号牌 | `5` |
| 出货号牌 | `あ` |
| 实际名称 | `张三` |

填好后点击 `新增登记`。新客户会自动加入永久客户簿。

如果是只买不卖的客户，出货号牌可以空着。如果是货主，一定要填出货号牌。
同一个客户在本场可以有多个出货号牌，但买货号牌必须是同一个。

## 5. 录入成交

进入 `现场录入`。

每件拍品填这几个字段：

| 字段 | 例子 | 说明 |
|---|---:|---|
| 拍品编号 | `1` | 第几件拍品 |
| 货主缩写 | `a` | 对应出货号牌 |
| 拍品缩写 | `hn` | 对应拍品名称 |
| 数量 | `1` | 件数 |
| 买家号牌 | `5` | 谁买了 |
| 千单位价 | `8` | 表示 8,000 |

确认后点击：

```text
保存拍品信息
```

## 6. 流拍怎么录

在 `现场录入` 填好拍品编号、货主缩写、拍品缩写和数量后：

1. 点击 `流拍`。
2. 买家号牌会变成 `-1`。
3. 点击 `保存拍品信息`。

## 7. 改错怎么做

如果刚录错了：

1. 在 `现场录入` 右侧 `最近录入` 找到那一行。
2. 点击 `改`。
3. 修改内容。
4. 点击 `保存修改`。

也可以到 `成交登记` 里找到那一行，点击 `改`。

## 8. 收款、付款、退货

进入 `成交登记`。

每行有三个按钮：

| 按钮 | 什么时候点 |
|---|---|
| `收` | 买家已经付款 |
| `付` | 已经给货主付款 |
| `退` | 流拍或顶价退回已经处理完 |

不要乱点：

1. 正常成交不要点 `退`。
2. 流拍或顶价退回不要点 `收`、`付`。
3. 点错了再点一次可以取消。

## 9. 看状态

重点看 `成交登记` 的状态。

| 状态 | 意思 |
|---|---|
| `未结算` | 买家还没付款 |
| `等待货主结算` | 买家已付，等付货主 |
| `已结算` | 收款和付款都完成 |
| `流拍待退回` | 流拍还没确认退回 |
| `顶价待退回` | 顶价退回还没确认 |
| `已退回` | 退回已完成 |
| `结算异常！` | 按钮点错了，要检查 |

## 10. 打精算传票

进入 `精算结算`。

1. 选择客户号牌。
2. 检查卖方明细和买方明细。
3. 看顶部金额：
   - `请款额`：客户还要付款。
   - `打款额`：需要打钱给客户。
4. 点击 `生成打印预览`。
5. 确认预览上方公司资料正确。
6. 点击 `打印`。

如果客户有多个出货号牌，可以在 `假名范围` 里只选其中一个。

## 11. 导出完整明细

进入 `完整明细`。

点击：

```text
下载 CSV
```

系统会下载：

```text
auction-full.csv
```

这个文件可用于对账、归档或发给外部人员。

## 12. 每天结束

按这个顺序做：

1. 到 `成交登记` 检查有没有 `结算异常！`。
2. 导出 `完整明细` CSV。
3. 备份 `auction-app/data/store.json` 和 `auction-app/data/sessions/`。
4. 关闭终端里的服务。

## 13. 出错先看这里

| 问题 | 怎么办 |
|---|---|
| 页面打不开 | 确认主机执行了 `npm start` |
| 其他电脑打不开 | 确认在同一 Wi-Fi，地址带 `http://` 和 `:4173` |
| 货主未匹配 | 检查货主缩写、出货号牌代码、客户资料 |
| 买家未匹配 | 检查买家号牌是否已登记 |
| 拍品未匹配 | 检查拍品缩写，或新增拍品代码 |
| 金额不对 | 确认千单位价没有填成完整金额 |
| 结算异常 | 检查 `收`、`付`、`退` 是否点错 |

## 14. 最小流程

```text
启动系统
-> 场次设置
-> 登记客户
-> 现场录入成交
-> 成交登记点收/付/退
-> 精算结算打印传票
-> 完整明细下载 CSV
-> 备份 store.json 和 sessions/
```
