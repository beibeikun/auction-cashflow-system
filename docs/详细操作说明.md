# auction-app 详细操作说明

本文档面向日常操作人员，按一场现金拍卖会的完整流程说明如何启动系统、准备数据、现场录入、确认收付款、处理退货、生成精算传票、导出明细和备份归档。

## 1. 系统角色

实际使用中可以按以下角色分工，也可以由一人完成全部操作。

| 角色 | 主要工作 |
|---|---|
| 主机操作员 | 启动服务、确认局域网地址、负责备份 |
| 现场录入员 | 使用 `现场录入` 快速登记拍品成交 |
| 收付款人员 | 使用 `成交登记` 确认收款、付款、退货 |
| 结算人员 | 使用 `精算结算` 打印或核对传票 |
| 管理员 | 维护客户资料、代码表、场次设置 |

## 2. 启动系统

在主机电脑打开终端：

```bash
cd auction-app
npm start
```

如果是第一次使用或刚拿到项目，先执行：

```bash
npm install
npm run build
npm start
```

启动成功后，终端会显示两个地址：

| 地址 | 用途 |
|---|---|
| `http://localhost:4173` | 主机本机访问 |
| `http://局域网IP:4173` | 其他设备访问 |

其他设备必须连接同一个 Wi-Fi 或同一局域网，然后用浏览器打开局域网地址。

## 3. 开始新场次前的准备

建议按顺序完成：

1. 备份上一场 `data/store.json` 和 `data/sessions/`。
2. 在 `场次设置 -> 拍卖场次管理` 新建空白场次，或切换到要继续处理的场次。
3. 如需从旧 Excel 开始，先切换到目标场次后执行 Excel 导入。
4. 检查 `场次设置`。
5. 检查 `客户资料`。
6. 检查 `拍品代码` 和 `出货号牌代码`。
7. 如本场要从空成交开始，在 `场次设置` 点击 `清空本场拍卖与客户数据`。

### 3.1 备份数据

数据文件是：

```text
auction-app/data/store.json
auction-app/data/sessions/*.json
```

建议复制一份并改名，例如：

```text
backup-20260503-1000/
```

### 3.2 从 Excel 导入

如果需要把旧 Excel 数据导入系统：

```bash
cd auction-app
python3 scripts/import_workbook_data.py "../docs/现金会v3.0.xlsx"
```

注意：Excel 导入会覆盖当前场次的 `data/sessions/<session-id>.json`，并更新全场次共享的拍品代码和出货号牌代码。请先在页面中切换到目标场次，并只在新场次开始前或确认可覆盖时执行。

### 3.3 清空本场拍卖与客户数据

进入：

```text
场次设置 -> 数据导入与清场 -> 清空本场拍卖与客户数据
```

这个操作会清空成交记录和本场客户登记，但保留：

1. 场次设置。
2. 全场次共享的拍品代码。
3. 全场次共享的出货号牌代码。
4. 永久客户簿。

## 4. 场次设置

进入 `场次设置`，确认以下内容。

| 字段 | 说明 |
|---|---|
| 拍卖会名称 | 显示在顶部、数据大屏和精算传票中 |
| 开拍时间 | 显示在顶部和数据大屏中 |
| 默认出货佣金 | 没有特殊货主佣金时使用 |
| 默认买货佣金 | 没有特殊买家佣金时使用 |
| 默认退货佣金 | 当前主要作为预留字段 |

填写规则：

1. 佣金只填数字，例如 5% 填 `5`。
2. 修改后点击 `保存设置`。
3. 在 `公司资料` 中维护税号、邮编、地址和公司 Logo。公司资料是所有场次共用，保存后会显示在精算传票打印预览中。
4. 保存后其他打开页面会自动刷新。

## 5. 维护拍品代码

拍品代码是所有场次共用的。任何场次中新增或修改后，切换到其他场次也会沿用同一套代码。

进入：

```text
场次设置 -> 拍品代码
```

每行格式：

```text
缩写,名称
```

示例：

```text
hn,花鳥
sf,書法
ss,山水
```

说明：

1. 现场录入时只输入缩写，例如 `hn`。
2. 系统显示时会转成完整名称，例如 `花鳥`。
3. 缩写建议使用小写字母或字母数字组合。
4. 修改完成后点击 `保存拍品代码`。

现场录入页也可以快速新增拍品名称：

1. 在 `快速创建新拍品名称` 输入名称。
2. 系统会自动生成缩写。
3. 点击 `加入拍品代码`。
4. 系统会把新缩写填入当前拍品表单。

## 6. 维护出货号牌代码

出货号牌代码是所有场次共用的。保存后会影响所有场次中“货主缩写 -> 出货号牌”的匹配关系。

进入：

```text
场次设置 -> 出货号牌代码
```

每行格式：

```text
英文缩写,日文号牌
```

示例：

```text
a,あ
i,い
u,う
```

说明：

1. 现场录入时输入英文缩写，例如 `a`。
2. 系统会通过 `a -> あ` 找到客户资料中的货主。
3. 如果成交记录显示货主未匹配，优先检查这里和客户资料是否一致。
4. 修改完成后点击 `保存号牌代码`。

## 7. 维护客户资料

进入 `客户资料`。

页面分为两类数据：

| 区域 | 用途 |
|---|---|
| 本场客户登记 | 维护本场使用的买货号牌、出货号牌和佣金 |
| 永久客户簿 | 长期保存客户实际名称、电话、地址、古物商证编号 |

同一个客户在本场只能对应一个买货号牌，但可以登记多个出货号牌。再次引用同一客户簿客户时，系统会自动沿用本场已有买货号牌。

### 7.1 从客户簿引用老客户

1. 在 `永久客户簿列表` 顶部使用 `搜索客户簿` 查找客户。
2. 可按客户簿 ID、实际名称、电话、地址或古物商证编号搜索。
3. 找到客户后点击左侧 `引用`。
4. 上方 `本场客户登记` 会自动带入客户簿资料。
5. 填写本场买货号牌、出货号牌和特殊佣金。
6. 点击 `新增登记`。

如果同一客户已经在本场登记过买货号牌，再新增出货号牌时会自动沿用原买货号牌；如果手动填了不同买货号牌，系统会提示冲突。

### 7.2 新登记客户

填写表单：

| 字段 | 说明 |
|---|---|
| 买货号牌 | 买家号牌，也是精算选择客户的编号 |
| 出货号牌 | 货主号牌，需要和出货号牌代码的日文号牌一致 |
| 实际名称 | 客户簿中的正式名称，页面、明细和精算传票使用 |
| 电话 | 客户簿联系方式 |
| 地址 | 客户簿地址 |
| 古物商证编号 | 客户簿证照编号 |
| 出货佣金 | 特殊货主佣金，空白则使用默认值 |
| 买货佣金 | 特殊买家佣金，空白则使用默认值 |
| 退货佣金 | 预留字段 |

填好后点击 `新增登记`。没有选择客户簿时，系统会自动生成客户簿 ID，并把实际名称、电话、地址和古物商证编号写入永久客户簿。

### 7.3 修改本场登记

1. 在右侧客户表中找到客户。
2. 点击 `改`。
3. 左侧表单会带出原数据。
4. 修改后点击 `保存登记`。

修改客户簿中的实际名称、电话、地址或古物商证编号，请在 `永久客户簿` 表单中编辑客户簿记录；客户簿资料是权威来源，改完后本场登记和结算显示会同步更新。

### 7.4 删除本场登记

1. 在客户表中点击 `删`。
2. 确认删除。

删除本场登记不会自动删除已有成交记录，也不会删除永久客户簿；相关成交可能出现买家或货主未匹配。

### 7.5 维护永久客户簿

在 `永久客户簿` 表单中可新增或修改：

1. 实际名称。
2. 电话。
3. 地址。
4. 古物商证编号。

永久客户簿列表中：

| 按钮 | 说明 |
|---|---|
| 引用 | 带入上方本场客户登记 |
| 改 | 编辑客户簿资料 |
| 删 | 删除未被本场登记引用的客户簿客户 |

### 7.6 流拍客户

系统需要保留：

```text
客户号牌 = -1
客户名称 = 流拍
```

买家号牌填 `-1` 时，系统会判断为 `流拍退回`。

## 8. CSV 导入

进入：

```text
场次设置 -> 数据导入与清场
```

操作步骤：

1. 点击文件选择框，选择 `.csv` 文件。
2. 如果只想更新本场客户登记，勾选 `只导入客户信息`。
3. 点击 `导入 CSV`。
4. 页面会显示导入客户数和成交数。

常用客户表头：

```text
客户号牌,出货号牌,客户名称,货主实际名称,客户电话,出货佣金比率,买货佣金比率,退货佣金比率
```

CSV 导入沿用旧表结构，只更新本场客户登记，不会自动写入永久客户簿。

常用成交表头：

```text
拍品编号,货主出货号牌缩写,拍品名称缩写,拍品数量,买家客户号牌,千单位成交价
```

## 9. 现场录入

进入 `现场录入`。

### 9.1 新增拍品成交

填写：

| 字段 | 示例 | 说明 |
|---|---:|---|
| 拍品编号 | `1` | 建议连续编号 |
| 货主缩写 | `a` | 必须在出货号牌代码中存在 |
| 拍品缩写 | `hn` | 必须在拍品代码中存在 |
| 数量 | `1` | 大于 3 时名称显示为 `山売` |
| 买家号牌 | `5` | 必须在客户资料中存在；流拍填 `-1` |
| 千单位价 | `8` | `8` 表示 `8,000` |
| 备注 | 可空 | 内部备注 |

确认无误后点击：

```text
保存拍品信息
```

保存后：

1. 记录进入最近录入列表。
2. 拍品编号自动跳到下一号。
3. 货主缩写和拍品缩写会记住上一次输入，方便连续录入。
4. 其他设备页面自动刷新。

### 9.2 修改指定拍品

方法一：

1. 在 `选择指定拍品修改` 下拉框选择拍品。
2. 修改表单。
3. 点击 `保存修改`。

方法二：

1. 在 `最近录入` 或 `成交登记` 中点击 `改`。
2. 系统切到 `现场录入`。
3. 修改并保存。

### 9.3 流拍

如果某件拍品流拍：

1. 填好拍品编号、货主缩写、拍品缩写、数量。
2. 点击 `流拍`。
3. 系统会把买家号牌设为 `-1`，价格为空时设为 `0`。
4. 点击 `保存拍品信息`。

保存后状态通常为：

```text
流拍待退回
```

退回完成后在成交登记中点击 `退`。

### 9.4 录入预警

现场录入表单下方会显示预览和警告。

| 警告 | 处理 |
|---|---|
| 货主缩写不存在 | 检查出货号牌代码和客户资料 |
| 拍品缩写不存在 | 检查拍品代码，或用快速创建新增 |
| 买家号牌未登记 | 先到客户资料新增本场登记 |

## 10. 数据大屏

进入 `数据大屏` 可以查看：

1. 当前正在录入的拍品。
2. 总成交额。
3. 总佣金额。
4. 买方应收。
5. 卖方应付。
6. 平均成交和最高成交。
7. 状态分布。
8. 最近成交。
9. 货主排行。

如果录入员正在填写但还没有保存，大屏会显示现场草稿。保存后显示正式成交。

## 11. 成交登记

进入 `成交登记`。

这里用于集中查看、搜索、筛选、批量确认和删除成交记录。

### 11.1 搜索和筛选

| 控件 | 说明 |
|---|---|
| 搜索框 | 可搜索号牌、客户、拍品等文本 |
| 全部号牌 | 可按客户号牌或出货号牌筛选 |
| 排序 | 可按拍品编号升序或降序 |
| CSV | 导出完整明细 CSV |

### 11.2 单条确认

每行有三个确认按钮：

| 按钮 | 含义 |
|---|---|
| `收` | 买家已付款或买家侧确认完成 |
| `付` | 已给货主付款或卖家侧确认完成 |
| `退` | 顶价退回或流拍退回已完成 |

点击按钮会在选中和未选中之间切换。

### 11.3 批量确认

操作步骤：

1. 勾选需要处理的成交行。
2. 或点击 `选择当前结果` 批量选择筛选后的结果。
3. 点击对应批量按钮。

批量按钮：

| 按钮 | 说明 |
|---|---|
| 批量收 | 选中行设置为买家已确认 |
| 取消收 | 选中行取消买家确认 |
| 批量付 | 选中行设置为卖家已确认 |
| 取消付 | 选中行取消卖家确认 |
| 批量退 | 选中行设置为退货已确认 |
| 取消退 | 选中行取消退货确认 |
| 删除所选 | 删除选中成交记录 |

### 11.4 删除记录

单条删除：

1. 点击行尾 `删`。
2. 确认删除。

批量删除：

1. 勾选多行。
2. 点击 `删除所选`。
3. 确认删除。

删除不可从页面撤销，删除前建议确认筛选条件和所选行。

## 12. 状态说明

| 状态 | 含义 | 常见处理 |
|---|---|---|
| `待拍` | 买家号牌为空 | 等待录入成交或流拍 |
| `未结算` | 正常成交但买卖双方都未确认 | 等待买家付款 |
| `等待货主结算` | 买家已确认，卖家未确认 | 可以给货主付款 |
| `已结算` | 买家和卖家都已确认 | 无需处理 |
| `货主已结算|买家未付款` | 卖家已确认但买家未确认 | 重点核对，避免先付后收 |
| `顶价待退回` | 买家号牌等于货主号牌 | 完成退回后点 `退` |
| `流拍待退回` | 买家号牌为 `-1` | 完成退回后点 `退` |
| `已退回` | 退货已确认 | 无需处理 |
| `结算异常！` | 确认按钮和业务类型冲突 | 检查 `收`、`付`、`退` 是否点错 |

异常规则：

1. 正常成交行不能点 `退`。
2. 顶价退回或流拍退回行不能点 `收` 或 `付`。
3. 如误点，取消对应按钮即可恢复。

## 13. 精算结算

进入 `精算结算`。

### 13.1 选择客户

1. 在 `客户号牌` 下拉框选择客户。
2. 如果该客户有多个出货号牌，可在 `假名范围` 中选择 `全部假名` 或某一个出货号牌。
3. 页面会显示卖方明细、买方明细和汇总金额。

### 13.2 金额含义

| 字段 | 说明 |
|---|---|
| 买方应收 | 该客户作为买家时需要支付的合计 |
| 卖方应付 | 该客户作为货主时应收到的合计 |
| 请款额 | 买方应收大于卖方应付，需要向客户收钱 |
| 打款额 | 卖方应付大于买方应收，需要向客户打款 |

计算方向：

```text
最终金额 = 买方应收 - 卖方应付
```

### 13.3 过滤选项

| 选项 | 说明 |
|---|---|
| 去除买 | 不看该客户作为买家的记录 |
| 去除卖 | 不看该客户作为卖家的记录 |
| 去除退货 | 排除顶价退回和流拍退回 |
| 只算退货 | 只看退回记录 |
| 全部显示 | 显示该客户全部相关记录，不只显示待处理记录 |
| 无视结算状态 | 金额计算包含全部相关记录，用于对账 |

### 13.4 打印精算传票

1. 选择客户号牌。
2. 检查假名范围。
3. 检查卖方明细和买方明细。
4. 检查请款额或打款额。
5. 点击 `生成打印预览`。
6. 确认传票顶部的公司 Logo、邮编、地址和税号正确。
7. 在预览层点击 `打印`。
8. 打印完成后点击 `关闭`。

## 14. 完整明细

进入 `完整明细`。

这里显示类似旧 Excel `完整信息` 的汇总表。

可以执行：

| 操作 | 说明 |
|---|---|
| 下载 CSV | 导出 `auction-full.csv` |
| 打印 | 打印当前完整明细页面 |

导出的 CSV 可用于归档、对账或发给外部人员。

## 15. 结场检查

拍卖结束前建议在 `成交登记` 中重点检查：

| 检查项 | 处理 |
|---|---|
| `结算异常！` | 必须先纠正 |
| `未结算` | 跟进买家付款 |
| `等待货主结算` | 跟进货主付款 |
| `货主已结算|买家未付款` | 高风险，必须核对 |
| `顶价待退回` | 确认是否已退回 |
| `流拍待退回` | 确认是否已退回 |
| `待拍` | 确认是否漏录成交或流拍 |

结场归档建议：

1. 导出完整明细 CSV。
2. 打印或保存需要的精算传票。
3. 复制备份 `data/store.json` 和 `data/sessions/`。
4. 记录本场备份文件名和时间。

## 16. 常见问题

### 其他设备打不开局域网地址

检查：

1. 主机是否执行 `npm start`。
2. 其他设备是否和主机在同一局域网。
3. 地址是否输入完整，例如 `http://192.168.1.10:4173`。
4. 主机防火墙是否允许 Node.js 或终端访问网络。

### 页面显示“正在连接”

可能原因：

1. 服务没有启动。
2. 网络断开。
3. 浏览器阻止了 SSE 长连接。

处理：

1. 刷新页面。
2. 确认 `npm start` 终端仍在运行。
3. 重新打开浏览器地址。

### 货主或买家显示未匹配

检查：

1. 客户是否已在 `客户资料` 的本场客户登记中登记。
2. 买家号牌是否填错。
3. 货主缩写是否在 `出货号牌代码` 中存在。
4. 出货号牌代码的日文号牌是否和客户资料的出货号牌一致。

### 拍品名称显示未匹配

检查：

1. 拍品缩写是否填错。
2. 拍品代码中是否存在该缩写。
3. 是否需要用 `快速创建新拍品名称` 新增代码。

### 金额看起来多了 1000 倍

系统沿用旧 Excel 的千单位价规则。

```text
千单位价 8 = 成交金额 8,000
```

如果成交金额应为 `8,000`，录入 `8` 是正确的；不要录入 `8000`。

### 出现结算异常

检查该行是否误点了确认按钮：

1. 正常成交行不要点 `退`。
2. 退回行不要点 `收` 或 `付`。
3. 取消误点按钮后状态会恢复。

### 导入 CSV 后成交数为 0

可能原因：

1. 勾选了 `只导入客户信息`。
2. CSV 表头不包含可识别的成交字段。
3. CSV 中拍品编号、货主缩写、拍品缩写都为空。

## 17. 安全提醒

1. `data/store.json` 和 `data/sessions/` 是业务数据源，务必定时备份。
2. 不要在多人操作时随意替换 `store.json` 或场次 JSON。
3. Excel 导入会覆盖当前场次数据。
4. 删除成交记录不可从页面撤销。
5. 清空本场拍卖与客户数据会删除全部成交记录和本场客户登记，但保留永久客户簿、共享代码和设置。
